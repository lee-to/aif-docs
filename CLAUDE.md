## Project Overview

Documentation site for the `lee-to/ai-factory` library, built on the Nuxt UI Docs template.

**Stack:** Nuxt 4, Nuxt Content v3, Nuxt UI 4.4, Tailwind CSS v4, TypeScript

## Commands

Use `bun`/`bunx` for all commands (translate any `npx` to `bunx`):

```bash
bun install          # Install dependencies
bun run dev          # Start dev server
bun run build        # Production build
bun run preview      # Preview production build
bun run lint         # ESLint
bun run typecheck    # TypeScript check (nuxt typecheck)
```

CI uses pnpm on Node 22 (lint + typecheck only).

## Architecture

### Content System (Nuxt Content v3)

Two collections defined in `content.config.ts`:
- **`landing`** — single `content/index.md`, rendered without prose (`<ContentRenderer :prose="false">`)
- **`docs`** — everything else under `content/`, rendered with full prose in the docs layout

Content files use numeric prefixes for ordering (`1.getting-started/`, `2.essentials/`). Nuxt Content strips prefixes from URLs. Each directory has a `.navigation.yml` for sidebar section titles/icons.

### Routing

- `app/pages/index.vue` — landing page, queries the `landing` collection
- `app/pages/[...slug].vue` — catch-all docs page, uses `docs` layout, queries `docs` collection by `route.path`

### Navigation Pattern

Navigation is fetched in `app/app.vue` via `useAsyncData('navigation', ...)` and provided to child components via Vue's `provide`/`inject`. The `error.vue` page also fetches navigation independently.

### Server Endpoints

- `/raw/[...slug].md` (`server/routes/raw/[...slug].md.get.ts`) — serves raw Markdown of any docs page
- MCP tools in `server/mcp/tools/` — `list-pages.ts` and `get-page.ts` expose docs via MCP protocol at `/mcp`
- `/llms.txt` and `/llms-full.txt` — auto-generated by `nuxt-llms` module

### Key Directories

- `app/components/content/` — MDC components usable inside Markdown files (e.g., `HeroBackground.vue`, `StarsBg.vue`)
- `app/components/OgImage/` — OG image templates for `nuxt-og-image`
- `app/assets/css/main.css` — global CSS: Tailwind v4 imports, Nuxt UI theme, custom green color palette, container width override

### Configuration Split

- `nuxt.config.ts` — modules, content settings, nitro prerender, eslint, llms/mcp config
- `app/app.config.ts` — UI theme colors, header/footer/toc links, SEO site name

## Nuxt Content Conventions

- Always refer to the skill at `.ai/skills/nuxt-content/SKILL.md` for Nuxt Content v3 patterns
- `queryCollection().path().first()` may return `null` — always handle it
- MDC components go in `app/components/content/`
- Frontmatter fields used in queries must be declared in the collection `schema` in `content.config.ts`
- Navigation icons use `i-lucide-*` or `i-simple-icons-*` prefix (Iconify)

## ESLint

Config extends Nuxt's generated config. Stylistic rules: `commaDangle: 'never'`, `braceStyle: '1tbs'`.
